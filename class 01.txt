단계별로 설명

1 클라이언트 side에서 패킷 발송 전 준비과정 / 2 클라이언트 구글 서버 사이의 패킷 전송

1) 클라이언트 side에서 패킷 발송 전 준비과정
DHCP를 이용해 client IP를 할당 == 이때, router ip와 DNS ip도 함께 받음

- 패킷 발송을 위해 소스ip, dest ip도 필요함으로, 
dns서버를 통해 구글 웹 서버의 ip를 획득 해야함

​
프레임이 라우터를 통해 DNS 서버로 (DNS서버로 요청이 포함된 프레임)을 전송하게 됨
​
우선 프레임이 라우터에 전달 되기 위해 라우터의 MAC 주소가 필요함
( - > 클라이언트에서 라우터 까지는 MAC주소를 통해 전달 하기 때문 )

라우터에서 DNS 서버 까지 경로는 OSPF, BGP 프로토콜에 따라 결정된 경로를 따름


2) 클라이언트 구글 서버 사이의 패킷 전송
1 클라이언트, 서버 간의 TCP connection을 맺어야 함
TCP Syn -> TCP SynACK -> ACK

2 이후 TCP connection이 맺어지면 그때부터 HTTP 요청 / 응답을 주고 받음
