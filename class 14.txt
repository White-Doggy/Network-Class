서버 프로세스 == 프로그램이 동작하며 진행되고있는 상태 상황

멀티 프로세스 / 멀티플렉싱 / 멀티 쓰레드



프로세스, 스레드 차이

멀티 프로세스 == 완전히 독립적으로 동일 프로그램을 복제해 자식 프로세스에서 처리
멀티 스레드 == 프로세스에서 다수의 스레드로 동작 일부 자원을 공유



멀티 플랙싱
하나의 프로세스에서 동작, 다수의 소켓에 대해 멀티플랙싱을 지원하는 함수를 통해
소켓이 특정 내용을 리턴하면 그때 해당 소켓에 접근하는 방식으로 다수의 접속을 처리

waitpid() 옵션 WNOHANG == 처리할 프로세스가 없어도 즉시 리턴



sigaction()
signal() 함수를 대체함, 특정 시그널에 대해 등록된 함수 동작을 실행

자식이 죽으면 SIGCHILD 라는 시그널이 호출되고 이를 sigaction을 통해
시그널 핸들러 내부에 wait()을 호출 == 좀비상태가 아닌 처리 되도록 함

이러한 과정을 통해 좀비 프로세스가 메모리를 차지하는 것을 막아 속도 저하를 막아야 함
