멀티 캐스트 & 브로드 캐스트

멀티캐스트 서버
특정 멀트캐스트 그룹 대상으로 데이터를 딱 한번 전송
한번 전송 해도 그룹 대상들은 모두 데이터를 수신
멀티캐스트 그룹의 수는 ip 주소 범위 내에서 추가 가능
데이터를 수신하려면 해당 그룹에 가입 필요
udp 소켓 기반



TTL (time to live)
패킷을 얼마나 멀리 보낼 것인가를 결정하는 요소

TTL은 정수, 라우터를 거칠 때 1 감소
TTL이 0이 되면 패킷 소멸

설정함으로써 길잃은 패킷을 줄일 수 있다.



그룹으로 가입방법
옵션 정보 변경을 통해 이루어짐, 그룹 가입에 활용되는 프로토콜 레벨 IPPROTO_IP
ip_mreq 구조체 변수 를 통해 표현 
imr_multiaddr == 가입할 멀티캐스트의 주소 
imr_interface == 본인의 주소

setsocket 함수를 통해 해당 구조체를 옵션으로 하여 그룹에 포함



브로드 캐스트
동일 네트워크 내에 존재하는 호스트에게 데이터 전송
호스트 대상이 아닌, 네트워크 기반
udp 소켓 기반
ip에서 네트워크 주소를 제외한 호스트 주소를 모두 1로 하여 전송하면 해당 네트워크로 전송됨 == directed 브로드 캐스트
255.255.255.255 로 데이터를 존성 시, 전송 호스트가 속한 네트워크로 데이터가 전송됨 == local 브로드 캐스트

소켓의 브로드 캐스트 옵션정보를 1로 변경해주면 됨
이후 주소 이외에는 달라지는 것이 없음
